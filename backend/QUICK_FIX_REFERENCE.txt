MOBILE APP 403 FORBIDDEN FIX - QUICK REFERENCE
================================================================================

ISSUE:
  Production Android users getting 403 Forbidden on sign-in/sign-up
  Error: POST /api/auth/sign-in/email → 403 Forbidden

ROOT CAUSE:
  Backend CORS configuration too restrictive for mobile apps

FIX APPLIED:
  Updated src/index.ts authentication configuration
  Changed: app.withAuth()
  To:      app.withAuth({ trustedOrigins: ["*"] })

STATUS: ✅ FIXED - READY FOR DEPLOYMENT

================================================================================
WHAT CHANGED
================================================================================

File: src/index.ts
Line: 67-72

Before:
  app.withAuth();

After:
  app.withAuth({
    trustedOrigins: ["*"],
  });

Impact: Mobile apps can now authenticate successfully
